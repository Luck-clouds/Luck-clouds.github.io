我的世界音乐包制作

话不多说

文件结构:总文件夹
├─assets
├─pack.mcmeta
├─pack.png
├─README.md
└─minecraft
├─lang
├─en_us.json
└─zh_cn.json
├─sounds.json
└─sounds
└─records
└─(music)

讲一下文件的各个用途

  1. 总文件夹:是整个音乐包所在，名字可以任意命名
  2. assets:二级文件夹
  3. pack.mcmeta:资源包的配置文件 有了它就能使游戏认识此文件
  4. pack.png:此资源包的封面(默认为原石)
  5. README.md:自述文件 可不写
  6. minecraft:三级文件夹
  7. lang:存放语言文件
  8. zh_cn.json:用于存放文字
  9. en_us.json:同上 内容也同上
  10. sounds.json:用于存放资源配置文件
  11. sounds:四级文件夹
  12. records:五级文件夹
  13. (music):用于存放音乐文件 ogg格式
  14. 渲染问题无法正常显示文件树 见谅

各配置文件的写法

pack.mcmeta

{
  "pack": {
    "pack_format": 8,
    "description": "\u00A76\u00A7b音乐唱片包-特供版\n\u00A77版本：1.2.2"
  }
}

由于json内部不允许添加注释 我在外面解释各词的用处

  1. pack_format :版本
  2. description :资源包描述 可以用ASCLL码转义更多花样
sounds.json

{
  "name": {
    "category": "record",
    "sounds": [
      {
        "name": "name",
        "stream": true
      }
    ],
    "subtitle": "描述"
  }
}
  1. name :填写records中的名字
  2. subtitle :游戏内资源描述
  3. 可以添加多行 但是要遵守json语法
  4. 此文件似乎会与lang文件夹及文件冲突 待考证
